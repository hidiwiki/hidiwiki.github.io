    一、基本概念

    SurfaceView是一个组件，可用于在View层次结构中嵌入其他合成层，与普通View不同，SurfaceView采用独立的线程进行渲染

    默认情况下，新创建的 Surface 放置在应用界面 Surface 的后面。可以替换默认的 Z 轴顺序，将新的 Surface 放在前面。

    当需要渲染特别复杂的图形时，使用SurfaceView会很有帮助，使用 SurfaceView 进行渲染时，SurfaceFlinger 会直接将缓冲区合成到屏幕上。如果没有 SurfaceView，需要将缓冲区合成到屏幕外的 Surface，然后该 Surface 会合成到屏幕上，而使用 SurfaceView 进行渲染可以省去额外的工作。

    二、SurfaceView的API总结

    2.1 SurfaceView使用步骤

    1. 首先要继承SurfaceView，实现SurfaceHolder.Callback接口如下

      private SurfaceHolder.Callback mCallback = new SurfaceHolder.Callback() {

    @Override
    public void surfaceCreated(@NonNull SurfaceHolder holder) {
        // Surface创建时触发，一般在这个函数开启绘图线程
        //（新的线程，不要再这个线程中绘制Surface）
        mIsDrawing = true;
        mThread.start();
    }

    @Override
    public void surfaceChanged(@NonNull SurfaceHolder holder, int format,
                               int width, int height) {
        // surface大小或格式发生变化时触发，在surfaceCreated调用后
        // 该函数至少会被调用一次
    }

    @Override
    public void surfaceDestroyed(@NonNull SurfaceHolder holder) {
        // 销毁时触发，一般不可见时就会销毁
    }
};

    2. 利用getHolder()获取SurfaceHolder对象，调用SurfaceHolder.addCallback添加回调

    3. SurfaceHolder.lockCanvas 获取Canvas对象并锁定画布，调用Canvas绘图

    4. SurfaceHolder.unlockCanvasAndPost 结束锁定画布，提交改变。

    SurfaceHolder
    SurfaceView的双缓冲的机制非常消耗系统内存，Android规定SurfaceView不可见时，会立即销毁SurfaceView的SurfaceHolder，以达到节约系统资源的目的，所以需要利用SurfaceHolder的回调函数对SurfaceHolder进行维护

    三、SurfaceView 和 Activity 生命周期

    当使用 SurfaceView 时，请使用主界面线程之外的线程渲染 Surface。

    对于具有 SurfaceView 的 Activity，存在两个单独但相互依赖的状态机：

    • 应用 onCreate/onResume/onPause
    • 已创建/更改/销毁的 Surface

    当 Activity 启动时，您将按以下顺序获得回调：

    1. onCreate()
    2. onResume()
    3. surfaceCreated()
    4. surfaceChanged()

    如果点击“BACK/HOME”，生命周期如下：

    1. onPause()
    2. surfaceDestroyed()（在 Surface 消失前调用）

    从home回到前台

    1. surfaceCreated
    2. surfaceChanged

    可以总结为，SurfaceView只要回后台，Surface就会被销毁

    四、SurfaceView的使用

    4.1 普通View的绘制

    普通View的绘制在onDraw()方法中进行的，时机是每次的Vsync，要使系统回调onDraw()方法，View自己要在合适的时机调用invalidata(),告诉系统该View加入回调队列，下一次Vsync到来时系统才会回调onDraw()方法，Vsync的频率一般为60HZ，因此，View的绘制必须要在16.7ms内完成，否则就会卡顿，出现跳帧现象，测试如下：

    每次点击屏幕，在点击位置绘制2000个同心圆，观察现象，代码如下：

    @Override
public void onTouchEvent(MotionEvent event) {
    centerX = event.getX();
    centerY = event.getY();
    invalidate();//标记下次Vsync调用onDraw回调
    return super.onTouchEvent(event);
}

@Override
public void onDraw(Canvas canvas) {
    super.onDraw(canvas);
    for (int i = 0; i < 2000; i++) {
        paint.color = xxx;
        canvas.drawCircle(centerX, centerY, i / 5, paint);
    }
}

    单次点击，绘制会造成明显延迟卡顿，且很容易发生ANR，截取的log如下：

    2022-03-22 12:01:43.798 10571-10593/com.iauto.surfaceview I/OpenGLRenderer: Davey! duration=1915ms; Flags=2, IntendedVsync=21900073767515, Vsync=21900073767515, OldestInputEvent=0, NewestInputEvent=0, ...

    从上面的log我们可以看出，存在丢帧的现象

    4.2 SurfaceView的绘制

    绘制同样的图形，使用SurfaceView绘制不会影响主线程的绘制

    多次点击后，发现上几次点击的图形不会被系统立刻清除，可以使用如下方式进行清空

    // 在绘制之前先绘制一次黑色
Canvas canvas = holder.lockCanvas();
canvas.drawColor(Color.BLACK);
// 之后再进行有效的绘制

    五、视频播放

    Android系统为我们准备了VideoView控件用于播放视频，该控件其实是封装了SurfaceView和MediaPlayer，功能相对简单，简单用法如下

    VideoView videoView = findViewById(R.id.videoView);
videoView.setVideoURI(Uri.parse("android.resource://"
                                + getPackageName() + "/" +R.raw.video));
// 显示控制条
videoView.setMediaController(new MediaController(this));
videoView.setOnPreparedListener(new MediaPlayer.OnPreparedListener() {
    @Override
    public void onPrepared(MediaPlayer mp) {
        // 控制播放属性
    }
});
videoView.start();

    使用该控件会有一个问题，Activity暂停再回到前台，无法实现无缝切换，会黑屏一段时间，时间长短取决于视频的加载速度，主要原因是，VideoView过早的释放了MediaPlayer，再次进入前台，重新创建一个MediaPlayer，所以会导致一段黑屏，这个问题我们使用SurfaceView来进行解决，方式可以有两个：

    • 方式一：重写VideoView，延迟MediaPlayer的释放时间

    • 方式二：将MediaPlayer放入一个ViewModel中进行维护

    我们采用方式二进行改进，ViewModel的生命周期比Activity的要长，不会随着屏幕旋转或退到后台而销毁，因此可以利用这一特性，用一个ViewModel来保存MediaPlayer，通过ViewModel来管理MediaPlayer的生命周期，sample code我已上传至github，请通过如下方式访问 ：传送门，另外还需要注意以下几点：

    • 实现一个MediaPlayer子类，继承自LifecycleObserver，然后和Activity绑定，即可实现回后台自动暂停，前台恢复播放

      public class MyMediaPlayer extends MediaPlayer implements LifecycleObserver {

    @OnLifecycleEvent(Lifecycle.Event.ON_PAUSE)
    public void pausePlayer() {
        pause();
    }

    @OnLifecycleEvent(Lifecycle.Event.ON_RESUME)
    public void resumePlayer() {
        if (!isPlaying()) {
            start();
        }
    }
}

    • 读取存储卡的权限问题，读取存储卡，我们一般在Manifest中申请如下两个权限：

      <uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>

      申请了以上两个权限，再去读取存储，发现还是没有权限，原因是这两个权限属于危险权限，必须要动态申请才可以
