    静态注册JNI函数

    一、静态注册的概念

    所谓的静态注册，就是根据函数名来找到对应的JNI函数，它需要Java的工具程序javah参与

    Java函数与JNI函数的对应关系

    如果Java类的包名为com.iauto.jni，则Java的函数的全路径名称为com.iauto.jni.HelloWorld.func()；在native中，"."有特殊的含义，因此，使用"_"代替，则Java的函数的全路径名称为com_iauto_jni_HelloWorld_func，下面我们实操一下。

    二、JAVA层的实现

    class HelloWorld
{
  private native void print();
  public static void main(String[] args)
  {
    new HelloWorld().print();
  }
  static{
    System.loadLibrary("libhelloworld_jni");// 此处有一个错误
  }
}

    三、创建一个JNI头文件

    Java根据函数名来查找Native 层的函数， 这就意味着Native层的函数名要和JAVA层的定义一致，如何做到一致呢？可以使用工具: javah, 使用方法如下

    // 1.首先使用javac将java文件转化为class文件
javac HelloWorld.java

// 2.使用javah自动生成一个HelloWorld.h
javah HelloWorld

    使用javah 时， 如果HelloWorld.java指定了包名，如在文件中指定com.iauto.jni

    package com.iauto.jni;

public class HelloWorld {
  ......
}

    如果指定了包名，则执行javah时，会报找不到文件的问题

    $ javah HelloWorld
错误: 找不到 'HelloWorld' 的类文件。

    解决方法如下：

    1. 假设项目的根目录为jni,在根目录下创建目录com/iauto/jni/目录，路径名称和包名一致，在该目录下创建HelloWorld.java

    2. 在根目录jni目录下创建src.txt文件（名称随意），填充内容如下

      com/iauto/jni/HelloWorld.java

      该目录用于指定java文件的路径，如果com/iauto/jni/目录下有多个java文件，可以在后面追加

    3. 在根目录jni目录下，创建class目录用于存放编译后的class文件，执行如下命令

      javac -d ./class @src.txt

      执行完毕后，会在class目录下生成HelloWorld.class文件，目录结构如下

      .
├── class
│   └── com
│       └── iauto
│           └── jni
│               └── HelloWorld.class

    4. 在根目录jni目录下，创建class.txt文件（名称随意），填充内容如下：

      com.iauto.jni.HelloWorld

    5. 在根目录jni目录下，创建jni目录（名称随意），然后在根目录jni下执行如下命令：

      javah -d jni/ -classpath class/ -jni @class.txt

      执行后，会在jni/jni/目录下，生成头文件com_iauto_jni_HelloWorld.h，目录结构如下：

      .
├── class
│   └── com
│       └── iauto
│           └── jni
│               └── HelloWorld.class
├── class.txt
├── com
│   └── iauto
│       └── jni
│           └── HelloWorld.java
├── jni
│   └── com_iauto_jni_HelloWorld.h
└── src.txt

8 directories, 6 files

    根据以上命令生成的com_iauto_jni_HelloWorld.h如下

    /* DO NOT EDIT THIS FILE - it is machine generated */
#include <jni.h>
/* Header for class com_iauto_jni_HelloWorld */

#ifndef _Included_com_iauto_jni_HelloWorld
#define _Included_com_iauto_jni_HelloWorld
#ifdef __cplusplus
extern "C" {
#endif
/*
 * Class:     com_iauto_jni_HelloWorld
 * Method:    print
 * Signature: ()V
 */
JNIEXPORT void JNICALL Java_com_iauto_jni_HelloWorld_print
  (JNIEnv *, jobject);

#ifdef __cplusplus
}
#endif
#endif

    四、实现Native函数

    #include "com_iauto_jni_HelloWorld.h"

JNIEXPORT void JNICALL Java_com_iauto_jni_HelloWorld_print
  (JNIEnv *, jobject) {
    printf("say hello world from jni !\n");
}

    如果java方法名中包含"_"，转化后的名称则以"_l"开头，如Java中的函数名为”native_print”，则转化后的方法名为"native_lprint"

    五、编译lib库

    首先使用gcc编译libmath.so 库

    gcc -fPIC -shared com_iauto_jni_HelloWorld.cpp -o libhelloworld_jni.so

    会报如下错误

    com_iauto_jni_HelloWorld.cpp:1:10: fatal error: jni.h: 没有那个文件或目录
    1 | #include <jni.h>
      |          ^~~~~~~
compilation terminated.

    原因就是没有配置环境变量，jni.h在/usr/lib/jvm/java-8-openjdk-amd64/include目录下，可以指定该目录，或者直接将其拷贝到jni/jni/目录下，同时把#include <jni.h> 变为 #include "jni.h"

    再次编译还是报错，如下：

    In file included from com_iauto_jni_HelloWorld.cpp:1:
jni.h:45:10: fatal error: jni_md.h: 没有那个文件或目录
   45 | #include "jni_md.h"
      |          ^~~~~~~~~~
compilation terminated.

    同理，需要把/usr/lib/jvm/java-8-openjdk-amd64/include/linux目录下的jni_md.h 拷贝到该目录下，再次编译OK

    六、运行Java程序

    在class目录下，执行如下命令，

    java com.iauto.jni.HelloWorld

    报如下错误：

    Exception in thread "main" java.lang.UnsatisfiedLinkError: no libhelloworld_jni.so in java.library.path
  at java.lang.ClassLoader.loadLibrary(ClassLoader.java:1860)
  at java.lang.Runtime.loadLibrary0(Runtime.java:871)
  at java.lang.System.loadLibrary(System.java:1124)
  at com.iauto.jni.HelloWorld.<clinit>(HelloWorld.java:11)

    原因是在java.library.path属性中找不到libhelloworld_jni.so库， 我们可以在java代码中，获取java.library.path属性，看一下都去哪些目录下查找

        String path = System.getProperty("java.library.path");
    System.out.println("path = " + path);

    Java运行时，会从如下目录中查找对应的库文件，

    path = /usr/java/packages/lib/amd64:/usr/lib/x86_64-linux-gnu/jni:/lib/x86_64-linux-gnu:/usr/lib/x86_64-linux-gnu:/usr/lib/jni:/lib:/usr/lib

    在运行命令时，通过-Djava.library.path来指定库目录，如下

    java com.iauto.jni.HelloWorld -Djava.library.path=/home/lhd/workspace/package/jni/jni/

    或者，通过LD_LIBRARY_PATH来指定

    export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/home/lhd/workspace/package/jni/jni/

    发现还是一如既往的报上面的错误，明明已经指定了lib库的目录，为什么就是找不到呢？肯定不是路径的错误，很有可能是库的名称的问题，再来看看我们是如何加载lib库的

      static {
    System.loadLibrary("libhelloworld_jni");
  }

    我们在一个静态代码块中，使用System.loadLibrary()方法进行库的加载，但是就是找不到libhelloworld_jni，原来，该方法在查找时，会自动补充一个lib前缀，如动态库的实际名称为libhelloworld_jni.so，那么System.loadLibrary()在实际查找时，就是liblibhelloworld_jni.so

    所有，我们应该这样修改, 去掉前面的lib前缀

      static {
    System.loadLibrary("helloworld_jni");
  }

    然后再运行就可以了

    $ java com.iauto.jni.HelloWorld -Djava.library.path=/home/lhd/workspace/package/jni/jni/
path = :/home/lhd/workspace/package/jni/jni/:.:/usr/java/packages/lib/amd64:/usr/lib/x86_64-linux-gnu/jni:/lib/x86_64-linux-gnu:/usr/lib/x86_64-linux-gnu:/usr/lib/jni:/lib:/usr/lib
say hello world from jni !

    七、javah命令

    使用javah -h命令，输出帮助

    用法: 
  javah [options] <classes>
其中, [options] 包括:
  -o <file>                输出文件 (只能使用 -d 或 -o 之一)
  -d <dir>                 输出目录
  -v  -verbose             启用详细输出
  -h  --help  -?           输出此消息
  -version                 输出版本信息
  -jni                     生成 JNI 样式的标头文件 (默认值)
  -force                   始终写入输出文件
  -classpath <path>        从中加载类的路径
  -cp <path>               从中加载类的路径
  -bootclasspath <path>    从中加载引导类的路径
<classes> 是使用其全限定名称指定的
(例如, java.lang.Object)。

    八、原理总结

    当Java层调用HelloWorld类中的print()的native方法时，它会从JNI库中查找Java_com_iauto_jni_HelloWorld_print()方法，如果没有，就会报错，如果找到，则会为print()方法和Java_com_iauto_jni_HelloWorld_print()方法建立一个连接，虚拟机会为其保存Java_com_iauto_jni_HelloWorld_print()的函数指针
