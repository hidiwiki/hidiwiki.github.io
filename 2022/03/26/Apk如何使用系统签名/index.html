    Apk如何使用系统签名

    一、概述

    两种方式，第一种方式：使用signapk.jar签名apk，第二种方式：使用Android Studio签名apk

    二、使用signapk.jar签名apk

    1. 为了后续方便操作，在源码根目录下新建一个文件夹signapk(名字任意)

      mkdir signapk

    2. 在源码中找到signapk.jar，将其拷贝到signapk目录下, signapk.jar的路径如下：

      android/prebuilts/sdk/tools/lib/signapk.jar

# 或者
android/out/host/linux-x86/framework/signapk.jar

    3. 在源码中找到平台签名文件platform.pk8和platform.x509.pem，路径如下(不同厂商路径不一样)：

      android/build/target/product/security/

    4. 把需要签名的APK放到signapk路径下

      android/signapk/app-debug.apk

      signapk的目录结构如下

      ./
├── app-debug.apk
├── platform.pk8
├── platform.x509.pem
└── signapk.jar

    5. 做完以上步骤后，如果在signapk路径下执行以下命令，会有问题，执行以下命令，这里有点坑，不能在signapk目录下，否则会报以下错误

      java -jar signapk.jar platform.x509.pem platform.pk8 app-debug.apk signapp.apk

      报错的原因是缺少conscrypt_openjdk_jni-linux-x86_64, conscrypt_openjdk_jni-linux-x86_64-fedora, conscrypt_openjdk_jni这几个包，这些包在prebuilts/sdk/tools/linux/lib64目录下，如果电脑的位数为32位，则需要引用prebuilts/sdk/tools/linux/lib,指定路径使用参数 -Djava.library.path，完整命令如下

      java -Xmx512m -Djava.library.path="prebuilts/sdk/tools/linux/lib64" -jar ./signapk/signapk.jar ./signapk/platform.x509.pem ./signapk/platform.pk8 ./signapk/app-debug.apk ./signapk/signapp.apk

      注意，这里有坑，一定不能在signapk目录下执行以上目标，不然会报段错误，原因未知

    6. 执行完以上命令后，如果不出意外，会在signapk目录下生成一个signapp.apk，该apk即为签名后的apk

    7. 如果在AndroidManifest.xml中未指定sharedUserId, 则apk为platform apk，如果想要apk为 system app， 则需要在AndroidManifest.xml中指定sharedUserId

      android:sharedUserId="android.uid.system"

    注意事项

    如果之前已经安装了未签名的app,需要先卸载，命令如下

    adb uninstall com.lhd.testapp

    三、使用Android Studio签名apk

    3.1 文件准备

    • /build/target/product/security/platform.pk8
    • /build/target/product/security/platform.x509.pem

    3.2 合成keystore

    为了调试开发方便我们需要将这两个文件生成为keystore文件，这里需要使用到一个工具keytool-importkeypair， 该工具可以把两个文件合成一个keystore文件

    把上面两个文件拷贝到keytool目录下，执行以下命令，

    ./keytool-importkeypair -k ~/.android/debug.keystore -p android -pk8 platform.pk8 -cert platform.x509.pem -alias platform

    命令执行完以后，就会在当前目录下生成一个debug.keystore文件

    • alias 别名 ： platform
    • -p 密码 ： android

    3.3 使用keystore文件签名

    为了方便，可以把上面的keystore文件拷贝到android studio 的project文件夹下，当然如果知道keystore路径，这一步可以不做

    打开Android studio, 点击File>Project Structure下，选择Modules选项> Signing Config，配置如下

    做完以上配置后，app的系统签名就完成了，此时，即可重新编译app，即为签名的app
