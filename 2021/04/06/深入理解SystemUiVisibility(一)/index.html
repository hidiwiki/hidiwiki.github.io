    深入理解SystemUIVisibility(一)

    SystemUiVisibility用于标记状态栏和导航栏的行为，如显示和隐藏，影响窗口的布局等，该标记由应用根据自己的需要设置给WMS，再有WMS综合考虑后通知SystemUI，依次来操作状态栏和导航栏

    一、SystemUIVisibility的分类

    按照这些标记所产生的影响，可以将其分为三类：

    影响状态栏与导航栏可见性的标记：

    • SYSTEM_UI_FLAG_LOW_PROFILE
    • SYSTEM_UI_FLAG_HIDE_NAVIGATION
    • SYSTEM_UI_FLAG_FULLSCREEN
    • SYSTEM_UI_FLAG_IMMERSIVE
    • SYSTEM_UI_FLAG_IMMERSIVE_STICKY

    影响窗口布局结果的标记：

    • SYSTEM_UI_FLAG_LAYOUT_STABLE
    • SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION
    • SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN

    用于禁用状态栏与导航栏的某些功能的标记：

    • STATUS_BAR_DISABLE_EXPAND
    • STATUS_BAR_DISABLE_NOTIFICATION__ICONS
    • STATUS_BAR_DISABLE_NOTIFICATION__ALERTS
    • STATUS_BAR_DISABLE_NOTIFICATION__TICKER
    • STATUS_BAR_DISABLE_INFO
    • STATUS_BAR_DISABLE_HOME
    • STATUS_BAR_DISABLE_BACK
    • STATUS_BAR_DISABLE_CLOCK
    • STATUS_BAR_DISABLE_RECENT
    • STATUS_BAR_DISABLE_SEARCH

    二、清除SystemUI的相关动作

    为了防止一些应用恶意将StatusBar 和 NavigationBar 一直隐藏下去，android系统定义了一些操作会打断StatusBar 和 NavigationBar的隐藏，这些操作可以分为以下几类：

    1. 触摸屏幕任意位置/按压任意按键
    2. 从屏幕顶部下拉状态栏/从屏幕底部下拉导航栏
    3. Window状态的变化（弹窗，输入法，跳转等）

    以上几类动作的容易触发程度从弱到强，配合SystemUI的Flag使用，防止恶意应用一直隐藏StatusBar和NavigationBar，对于第二类操作，如无特殊说明，后文在描述StatusBar 时，指的是从屏幕顶部下拉状态栏， 描述NavigationBar时，指的是从屏幕底部下拉导航栏

    三、SystemUI Flag 相关特性详解

    3.1 SYSTEM_UI_FLAG_LOW_PROFILE

    作用：使状态栏与导航栏进入低调模式

    @Override
protected void onResume() {
    super.onResume();
    int systemUiFlag = getWindow().getDecorView().getSystemUiVisibility();
    int flag = systemUiFlag | View.SYSTEM_UI_FLAG_LOW_PROFILE;
    getWindow().getDecorView().setSystemUiVisibility(flag);
}

    低调模式设置后status bar上的一些图标会不显示，navigation bar没有变化，效果见下图：

    3.2 SYSTEM_UI_FLAG_HIDE_NAVIGATION

    作用：隐藏导航栏，窗口在布局时的ParentFrame,ContentFrame，以及DisplayFrame时，会延伸至整个屏幕

    @Override
protected void onResume() {
    super.onResume();
    int systemUiFlag = getWindow().getDecorView().getSystemUiVisibility();
    int flag = systemUiFlag | View.SYSTEM_UI_FLAG_HIDE_NAVIGATION;
    getWindow().getDecorView().setSystemUiVisibility(flag);
}

    设置该Flag后，用户与系统的任何交互（第一、二、三类操作）都会导致系统将其清除，进而导航栏会重新显示，设置后的效果如下：

    3.3 SYSTEM_UI_FLAG_FULLSCREEN

    作用：隐藏StatusBar,从定义来看，是进入全屏模式，但是该Flag只能隐藏StatusBar，并不能隐藏NavigationBar

    @Override
protected void onResume() {
    super.onResume();
    int systemUiFlag = getWindow().getDecorView().getSystemUiVisibility();
    int flag = systemUiFlag | View.SYSTEM_UI_FLAG_FULLSCREEN;
    getWindow().getDecorView().setSystemUiVisibility(flag);
}

    设置该Flag后，用户的交互（跳转到其他应用,下拉StatusBar，弹出键盘等,也即是第二章定义的第二、三类操作）会打断StatusBar的隐藏，设置后的效果如下：

    3.4 SYSTEM_UI_FLAG_IMMERSIVE

    作用:提高用户操作的等级，避免某些用户交互造成系统自动清除全屏状态，

    该Flag只配合SYSTEM_UI_FLAG_HIDE_NAVIGATION使用，对StatusBar的状态不受影响

    设置该Flag后，只有第二、三类操作才会导致系统清除SYSTEM_UI_FLAG_HIDE_NAVIGATION

    3.5 SYSTEM_UI_FLAG_IMMERSIVE_STICKY

    作用：（1）提高用户操作的等级，（2）隐藏状态以半透明状态显示，延迟一段时间后会被重新隐藏，（3）NavigationBar显示后，会遮挡当前的画面，当前画面不会重新绘制

    @Override
protected void onResume() {
    super.onResume();
    int systemUiFlag = getWindow().getDecorView().getSystemUiVisibility();
    int flag = systemUiFlag 
        | View.SYSTEM_UI_FLAG_HIDE_NAVIGATION
        | View.SYSTEM_UI_FLAG_IMMERSIVE_STICKY;
    getWindow().getDecorView().setSystemUiVisibility(flag);
}

    此Flag只有配合SYSTEM_UI_FLAG_FULLSCREEN和SYSTEM_UI_FLAG_HIDE_NAVIGATION使用时才会起作用。

    设置效果如下：

    3.6 SYSTEM_UI_FLAG_LAYOUT_STABLE

    作用：该标记不会对状态栏和导航栏的可见性产生影响，会对声明这个Flag的窗口的布局产生影响

    不设置该属性，StatusBar和NavigationBar显示和隐藏时，会使得ContentFrame不断变化，进而使得控件所绘制的内容上下跳动，用户的实际感觉是屏幕一闪

    声明该属性后，状态栏在显示和隐藏时不会对窗口的绘制产生影响，看起来非常的平滑

    同时设置了SYSTEM_UI_FLAG_FULLSCREEN | SYSTEM_UI_FLAG_LAYOUT_STABLE时，会同时隐藏Actionbar和StatusBar，StatusBar所占空间不会隐藏，只会变成空白。同时View所在window的显示范围也不会伸展到StatusBar所占空间

    @Override
protected void onResume() {
    super.onResume();
    int systemUiFlag = getWindow().getDecorView().getSystemUiVisibility();
    int flag = systemUiFlag
        | View.SYSTEM_UI_FLAG_FULLSCREEN
        | View.SYSTEM_UI_FLAG_IMMERSIVE
        | View.SYSTEM_UI_FLAG_LAYOUT_STABLE;
    getWindow().getDecorView().setSystemUiVisibility(flag);
}

    设置后的效果如下，Navigationbar同理

    3.7 SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION

    作用：在不隐藏导航栏的情况下，设置该属性的窗口的显示内容将延伸到NavigationBar底部，同时窗口的内容也因为NavigationBar而覆盖

    在使用SOFT_INPUT_ADJUST_RESIZE，

    同时View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN或View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION时，当键盘弹出时，只会fitSystemWindow=true的view所占区域会被resize，其他view将会被软键盘覆盖。

    3.8 SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN

    与SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION作用相同，只不过该属性作用与StatusBar

    四、WindowManager 相关特性详解

    与StatusBar 和 NavigationBar 状态相关的属性，除了在View中定义的以上属性，系统在WindowManager也定义了其他的都属性，用于控制两个bar的状态

    • WindowMananger.FLAG_TRANSLUCENT_STATUS
    • WindowMananger.FLAG_FULLSCREEN
    • WindowMananger.FLAG_TRANSLUCENT_NAVIGATION
    • WindowMananger.FLAG_DRAWS_SYSTEM_BAR_BACKGROUNDS

    五、常用组合

    一般设置SystemUIVisibility的时机有两个：

    protected void onResume();

    public void onWindowFocusChanged(boolean hasFocus)

    5.1 设置全屏模式

    @Override
protected void onResume() {
    super.onResume();
    int systemUiFlag = getWindow().getDecorView().getSystemUiVisibility();
    int flag = systemUiFlag
        | View.SYSTEM_UI_FLAG_FULLSCREEN
        | View.SYSTEM_UI_FLAG_HIDE_NAVIGATION
        | View.SYSTEM_UI_FLAG_LAYOUT_FULLSCREEN;
        | View.SYSTEM_UI_FLAG_LAYOUT_HIDE_NAVIGATION
        | View.SYSTEM_UI_FLAG_LAYOUT_STABL;
    getWindow().getDecorView().setSystemUiVisibility(flag);
}
