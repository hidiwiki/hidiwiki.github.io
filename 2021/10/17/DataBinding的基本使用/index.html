    DataBinding的基本使用

    一、基本概念

    下面通过一个示例来说明DataBinding的作用

    一个简单的示例：画面内有一个Button和一个TextView，每次点击Button，TextView数字加1，布局如下：

    二、传统实现

    使用ViewModel来存放界面的数据，ViewModel的代码如下

    public class MainViewModel extends ViewModel {
    private MutableLiveData<Integer> count;

    public MutableLiveData<Integer> getCount() {
        if (count == null) {
            count = new MutableLiveData<>();
            count.setValue(0);
        }
        return count;
    }

    public void addCount() {
        count.setValue(count.getValue() + 1);
    }
}

    在MainActivity中，每次点击Button，count值加1，同时需要更新TextView的值，代码如下：

    public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        Button button = findViewById(R.id.button);
        TextView view = findViewById(R.id.textView);
        MainViewModel model = new ViewModelProvider(this).get(MainViewModel.class);
        model.getCount().observe(this, new Observer<Integer>() {
            @Override
            public void onChanged(Integer integer) {
                view.setText(String.valueOf(integer));
            }
        });
        button.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                model.addCount();
            }
        });
    }
}

    从上面的代码中我们可以发现，MainActivity的代码非常的不方便维护，如界面内有多少个控件，我们就需要使用多少次findViewById()去创建控件，而且还要去设置相应的监听，下面我们使用databinding进行优化

    三、使用dataBinding优化

    首先，将布局文件转换为databinding布局，

    <?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"

    android:layout_height="match_parent"

        android:layout_height="wrap_content"

        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toTopOf="parent" />

    <TextView
        android:id="@+id/textView"

        android:layout_height="wrap_content"

        app:layout_constraintEnd_toEndOf="parent"
        app:layout_constraintStart_toStartOf="parent"
        app:layout_constraintTop_toTopOf="parent"
        app:layout_constraintVertical_bias="0.374" />

</androidx.constraintlayout.widget.ConstraintLayout>

    在转换成databinding布局之前，首先需要在gradle中设置了如下属性

    dataBinding {
    enabled = true
}

    转换成databinding布局之后代码如下

    </androidx.constraintlayout.widget.ConstraintLayout>

    以前的布局被layout标签包含，同时在layout标签内增加data标签用于声明MainActivity画面内的数据

    <data>

</data>

    在进行转换后，整个布局文件会被IDE在后台自动解析成一个类文件，其中类文件的名称为布局文件的名称+Binding，布局文件名称需要去掉下划线，同时采用驼峰命名法即每个单词的首字母大写，以activity_main.xml为例，则生成的类文件为ActivityMainBinding.java，在MainActivity中我们应该这样使

    public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        MainViewModel model = new ViewModelProvider(this).get(MainViewModel.class);

        ActivityMainBinding binding = DataBindingUtil.setContentView(this, R.layout.activity_main);

        model.getCount().observe(this, new Observer<Integer>() {
            @Override
            public void onChanged(Integer integer) {
                binding.textView.setText(String.valueOf(integer));
            }
        });
        binding.button.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                model.addCount();
            }
        });
    }
}

    改变如下：

    • TextView和Button不再通过findViewById()的方式，而是通过DataBindingUtil创建一个ActivityMainBinding对象，通过此对象获取这两个View
    • MainActivity内的所有用到TextView和Button的地方都换成从ActivityMainBinding中获取

    以上的改变似乎并没有多少优化的空间，别急，还记得上文中提到的布局文件中增加的data标签么？，我们可以在data标签内进行如下的声明

    <data>
    <variable
          name="modelData"
          type="com.iauto.mvvm.MainViewModel" />
</data>

    在data标签内，我们声明了一个变量modelData，类型为com.iauto.mvvm.MainViewModel，接着我们把TextView要显示的内容与modelData中的count进行绑定

    <TextView    ......    android:text="@{String.valueOf(modelData.count)}"    ....../>

    这里说明一下：使用@{}表示将TextView的text值与modelData的count进行单项绑定，即当modelData.count值发生变化时，会通知TextView进行刷新，同时还需要把modelData.count转成String类型，同理，也可以为Button的onClick()方法绑定到modelData的addCount()方法上

        <Button
        ......
        android:onClick="@{()->modelData.addCount()}"
        ......
    />

    表示当Button的onClick()方法被回调时，onClick()方法中会调用modelData.addCount()，在对布局文件进行了以上的优化后，我们就可以继续改造MainActivity的处理

    public class MainActivity extends AppCompatActivity {    @Override    protected void onCreate(Bundle savedInstanceState) {        super.onCreate(savedInstanceState);        setContentView(R.layout.activity_main);        MainViewModel model = new ViewModelProvider(this).get(MainViewModel.class);        ActivityMainBinding binding = DataBindingUtil.setContentView(this, R.layout.activity_main);        binding.setModelData(model);        binding.setLifecycleOwner(this);    }}

    以前设置的各种监听就不需要了，因为在布局文件中，已经把控件和数据进行了单向的绑定，由于在布局中声明了类型为com.iauto.mvvm.MainViewModel的modelData变量，在实例化之后就需要调用setModelData()给该变量赋值

    总结

    经过以上的优化，MainActivity的逻辑是不是少了很多，就达到了便于维护的目的，以上我们只使用了单向绑定，其实也可以进行双向绑定，使用@={}即可进行双向绑定，对于大多数控件，android已经为我们实现了单向和双向的绑定，但对于自定义的View需要我们自己实现，后面的文章会继续分析
